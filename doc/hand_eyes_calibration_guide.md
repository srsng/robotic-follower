## 项目概述

`hand_eyes_calibration` 是一个机器人手眼标定（Hand-Eye Calibration）工具包，用于工业视觉与机器人协同开发中的精确标定。

### 主要功能

| 功能类型 | 说明 | 适用场景 |
|---------|------|---------|
| **相机内参标定** | 计算相机焦距、主点、畸变系数等内参 | 所有标定前必需步骤 |
| **2D 手眼标定** | 像素坐标到机器人工作空间的平面转换 | 2D 相机抓取定位 |
| **3D 手眼标定** | 相位坐标系到机器人坐标系的 6D 变换 | 3D 相机深度信息利用 |
| **实时检测** | ArUco 标记检测与 6DoF 位姿转换 | 验证标定结果 |

### 支持的标定方式

1. **内参标定（Intrinsic Calibration）**
2. **2D 4点插针标定（Four-Point Pin）**
3. **2D 12点标定（Twelve-Point Calibration）**
4. **3D 标定板图像标定（Board Image）**
5. **3D 标定球标定（Ball-based）**
6. **3D 标定板图像+点云标定（Board Image + Cloud）【实验性】**

### 支持的机器人

- KUKA, Fanuc, ABB, UR（Universal Robots）等主流工业机器人
- 支持全部 12 种欧拉角旋转顺序（内旋/外旋）
- UR 机器人支持旋转向量（Axis-Angle）

### 在 `calib.cpp` 中通过宏开关选择标定任务

```cpp
// 内参标定
#define RUN_INTRINSIC 1

// 2D 4点标定
#define RUN_HAND_EYE_2D_4POINT 1

// 2D 12点标定
#define RUN_HAND_EYE_2D_12POINT 1

// 3D 标定板图像标定
#define RUN_HAND_EYE_3D_BOARD_IMG 1

// 3D 标定球标定
#define RUN_HAND_EYE_3D_BALL 1

// 3D 标定板图像+点云标定（实验性）
#define RUN_HAND_EYE_3D_BOARD_CLOUD 0
```

### 如何选择标定任务

1. 将对应宏定义为 `1` 表示启用该标定任务
2. 将对应宏定义为 `0` 表示禁用该标定任务
3. 修改后需要重新编译：`cd build && make -j8`

## 详细标定流程

## 一、通用准备工作

### 1. 标定板准备

| 标定板类型 | 推荐规格 | 打印要求 |
|-----------|---------|---------|
| 非对称圆点 | 4×5，间距 20mm | 需用卡尺测量实际间距 `interval_mm` |
| 棋盘格 | 7×6，间距 25mm | 标准 A4 打印即可 |

**注意事项**：
- 打印后必须用卡尺测量实际间距 `interval_mm`
- 确保打印平整，无变形
- 标定板应粘贴在刚性板上

### 2. 机器人位姿记录约定

所有需要机器人位姿的标定都需要同时记录：
- **位姿数据**：`(x, y, z, rx, ry, rz)` 单位 mm 和度
- **欧拉角约定**：根据机器人品牌设置

| 机器人品牌 | 欧拉角顺序 |
|-----------|-----------|
| KUKA | ZYX 外旋 |
| Fanuc | XYZ 外旋 |
| UR | ROT_VECTOR（旋转向量） |

## 二、1. 内参标定（Intrinsic Calibration）

### 设备要求
- **相机**：任意相机（工业相机、USB相机等）

### 拍摄方法

1. 手持标定板或固定相机后移动标定板
2. 在**不同角度、不同距离**拍摄 15-20 张图片
3. 确保标定板板覆盖整个图像视野

### 拍摄注意事项

| ✅ 推荐做法 | ❌ 避免做法 |
|-----------|-----------|
| 标定板必须完整出现在图像中 | 标定板边缘被裁剪 |
| 拍摄角度要有足够变化（至少 40-60 度旋转） | 所有图像角度相近 |
| 避免强光直射，防止反光 | 强光或镜面反射 |
| 图像清晰，无运动模糊 | 图像模糊或抖动 |
| 避免标定板产生阴影 | 阴影遮挡标定板 |

### 数据提供方式

将拍摄的图像保存为 `.png` 或 `.jpg` 格式，放在 `assert/camera/` 目录下，然后在 `calib.cpp` 中修改文件列表：

```cpp
std::vector<std::string> intrinsic_img_files = {
    "../assert/camera/10.png", "../assert/camera/11.png",
    "../assert/camera/12.png", "../assert/camera/13.png",
    // ... 添加更多图片
};
```

### 配置标定板参数（在 `calib.cpp` 中）

```cpp
// 非对称圆点（推荐）
cfg.pattern = CIRCLES_ASYM;
cfg.cols = 4;
cfg.rows = 5;
cfg.interval_mm = 20;  // 标定板圆心间距（实际测量值）

// 或者使用棋盘格
cfg.pattern = CHESSBOARD;
cfg.cols = 7;
cfg.rows = 6;
cfg.interval_mm = 25;  // 标定板方格间距（实际测量值）
```

### 运行标定

```bash
cd build
./calib
```

### 输出结果

- **内参文件**：`calib_intrinsic_1st.xml`
- **关键参数**：
  - 相机内参矩阵 `K`
  - 畸变系数 `distortion`
  - 重投影误差（RMS）

### 质量评估标准

| 指标 | 合格标准 |
|-----|---------|
| RMS 误差（重投影误差） | < 0.2 像素 |

## 三、2. 2D 4点插针标定（Four-Point Pin）

### 设备要求
- **相机**：2D 相机（固定在机器人或外部支架上）

### 拍摄方法

1. 准备 4 个**已知物理坐标**的标记点（如工作台上的 4 个定位孔）
2. 移动机器人到每个标记点，记录机器人坐标和对应的图像像素坐标
3. **只需拍摄一张图片**，记录 4 个特征点的位置

### 拍摄注意事项

| ✅ 推荐做法 | ❌ 避免做法 |
|-----------|-----------|
| 相机与工作台距离必须**恒定** | 相机高度变化 |
| 4 个点应形成**矩形布局**，避免共线 | 4 点共线或排列不规则 |
| 点之间距离尽可能大，以提高精度 | 点距离过近 |
| 所有点在同一平面内 | 点不在同一平面 |

### 数据记录格式

需要记录两份数据：
1. **图像像素坐标**（4个点）：`(u, v)`
2. **机器人物理坐标**（4个点）：`(x, y)` 单位 mm

### 数据提供方式（在 `calib.cpp` 中）

```cpp
// 图像像素坐标（4个点）
std::vector<cv::Point2d> img_points = {
    {2035, 1093}, {3129, 1087}, {3130, 1549}, {2038, 1556}
};

// 机器人物理坐标（4个点）
std::vector<cv::Point2d> robot_points = {
    {800.94, 886.14}, {805.45, 570.15}, {671.78, 568.56}, {667.07, 884.8}
};
```

### 运行标定

```bash
cd build
./calib
```

### 输出结果

- **外参文件**：`calib_extrinsic.xml`
- **转换矩阵**：像素坐标 → 机器人工作空间坐标

### 质量评估标准

| 指标 | 合格标准 |
|-----|---------|
| 像素误差 | < 0.5 像素 |

## 四、3. 2D 12点标定（Twelve-Point Calibration）

### 设备要求
- **相机**：2D 相机

### 拍摄方法

**此标定需要两组数据：**

#### 第一组：9 点仿射变换（平面映射）

1. 在工作面上标记或识别一个**注册点**（参考点）
2. 以注册点为中心，在 8 个方向移动机器人（形成十字形或网格）
3. 在每个位置拍摄一张图片，记录机器人位姿和图像中的标记点位置

#### 第二组：3 点旋转拟合

1. 在注册点位置，让机器人**绕 Z 轴旋转**（如 -10°、0°、10°）
2. 拍摄 3 张图片，记录旋转角度和标记点在图像中的运动轨迹

### 拍摄注意事项

| ⚠️ 非常重要 | 说明 |
|---------|------|
| 相机与物体的拍照距离必须保持**恒定** | 否则标定失败 |

| ✅ 推荐做法 | ❌ 避免做法 |
|-----------|-----------|
| 前 9 点应在同一平面内移动 | Z 轴高度变化 |
| 旋转时标记点在图像中应画出明显圆弧 | 旋转角度太小 |
| 注册点应清晰可识别 | 注册点模糊或遮挡 |

### 数据记录格式

需要记录：
1. **机器人位姿**（12个）：`(dx, dy, rz)` 相对注册点的偏移
2. **图像标记点坐标**（12个）：`(x, y, theta)` 像素坐标和角度

### 数据提供方式（在 `calib.cpp` 中）

```cpp
// 注册点在图像中的位置
Pose2D registration_point = {1188, 468, 0};

// 12个图像点
std::vector<Pose2D> img_points_12p = {
    {1188, 468, -0}, {1173, 467, -0},
    // ... 共12个图像点
};

// 12个机器人位姿
std::vector<Pose2D> robot_poses_12p = {
    {0, 0, 0}, {0, 30, 0}, {30, 30, 0},
    // ... 共12个机器人位姿
};
```

### 运行标定

```bash
cd build
./calib
```

### 输出结果

- **外参文件**：`calib_extrinsic.xml`
- **高精度转换矩阵**：考虑旋转补偿

### 质量评估标准

| 指标 | 合格标准 |
|-----|---------|
| 像素误差 | < 0.5 像素 |

## 五、4. 3D 标定板图像标定（Board Image）

### 设备要求
- **相机**：3D 深度相机（如 Realsense D435、Intel L515 等）
- 可用 `detect.cpp` 或相机 SDK 实时采集

### 拍摄方法

1. **眼在手上**：相机安装在机器人末端，移动机器人到不同位置拍摄
2. **眼在手外**：相机固定在工作台上，移动标定板或机器人末端工具

**推荐拍摄策略**：
- 采集 **15-25 张**不同位姿的图片
- 覆盖相机视野的**前、后、左、右、上、下、近、远** 8 个方向
- 每张图片都包含**较大角度的旋转**（±30°以上）
- 位姿变化应具有**随机性**，避免规律性移动

### 拍摄注意事项

| ⚠️ 非常重要 | 说明 |
|-----------|------|
| **必须包含旋转**！纯平移无法准确标定旋转矩阵 | 旋转对标定质量至关重要 |

| ✅ 推荐做法 | ❌ 避免做法 |
|-----------|-----------|
| 标定板必须完整出现在图像中 | 标定板边缘被裁剪或遮挡 |
| 每张图像都要有明显的旋转变化 | 纯平移移动 |
| 保持深度相机工作距离有效（通常 0.5m - 3m） | 距离超出相机有效范围 |
| 环境光均匀，避免强阴影 | 强阴影或反光 |

### 数据记录格式

每张图片需要对应一组数据：
- **图像文件**：标定板照片
- **机器人位姿**：`(x, y, z, rx, ry, rz)`

### 数据提供方式（在 `calib.cpp` 中）

1. 将图像保存到 `assert/camera/` 目录
2. 配置图像文件列表：

```cpp
std::vector<std::string> extrinsic_img_files = {
    "../assert/camera/10.png", "../assert/camera/11.png",
    // ... 每个图像文件
};
```

3. 配置对应的机器人位姿：

```cpp
const std::vector<Pose3D> robot_poses = {
    {211.892, -415.103, -536.213, -91.135, 44.296, 11.726},
    {217.353, -412.223, -530.437, -89.057, 44.896, 13.651},
    // ... 每个图像对应的机器人位姿
};
```

4. 配置欧拉角类型（根据机器人品牌）：

```cpp
// 示例：XYZ 外旋（适用于 Fanuc 等机器人）
board_calib_obj.setRPYType(RPY::RPYType::XYZ, RPY::ReferenceType::EXTRINSIC);

// 示例：ZYX 外旋（适用于 KUKA 等机器人）
board_calib_obj.setRPYType(RPY::RPYType::ZYX, RPY::ReferenceType::EXTRINSIC);
```

### 运行标定

```bash
cd build
./calib
```

### 输出结果

- **外参文件**：`calib_extrinsic.xml`
- **手眼变换矩阵**：`T_hand_eye`
- **关键参数**：
  - 旋转矩阵 `R`
  - 平移向量 `t`
  - RMS 误差（重投影误差）
  - 平均物理误差

### 质量评估标准

| 指标 | 合格标准 |
|-----|---------|
| RMS 误差 | < 0.3 像素 |
| 平均物理误差 | 0.1-1.0 mm |

## 六、5. 3D 标定球标定（Ball-based）

### 设备要求
- **相机**：3D 深度相机
- **标定球**：标准陶瓷或金属标定球（直径已知）

### 拍摄方法

1. 将标定球固定在工作台上或放置在可移动平台上
2. 使用点云处理算法提取球心坐标 `(x, y, z)`
3. 同时记录相机位置时对应的机器人位姿
4. 在 **不同角度和距离**采集 10-20 组数据

### 拍摄注意事项

| ✅ 推荐做法 | ❌ 避免做法 |
|-----------|-----------|
| 标定球表面应均匀，避免局部反光 | 标定球表面镜面反射 |
| 点云中球体部分应完整 | 球体被遮挡 |
| 避免背景干扰 | 背景中类似球体的物体 |
| 球心提取精度直接影响标定结果 | 球心提取不准确 |

### 数据记录格式

每组数据包含：
- **机器人位姿**：`(x, y, z, rx, ry, rz)`
- **球心坐标**：`(x, y, z)` 在相机坐标系下，单位 mm

### 数据提供方式（在 `calib.cpp` 中）

无需拍摄图像，直接在代码中输入球心坐标：

```cpp
// 机器人位姿数据
std::vector<std::vector<double>> robot_poses_data = {
    {-1223.07, 1493.14, 774.351, -172.691, -14.074, 170.725},
    // ... 更多机器人位姿
};

// 对应的球心坐标数据
std::vector<std::vector<double>> marker_points_data = {
    {-107, -2.00, 632.75}, {-3.75, -39.00, 528},
    // ... 更多球心坐标
};
```

### 运行标定

```bash
cd build
./calib
```

### 输出结果

- **外参文件**：`calib_extrinsic.xml`
- **手眼变换矩阵**

### 质量评估标准

| 指标 | 合格标准 |
|-----|---------|
| 物理误差 | 2-5 mm（受光照影响较大） |

## 七、6. 3D 标定板图像+点云标定（Board Image + Cloud）【实验性】

### 设备要求
- **相机**：3D 深度相机（需获取图像和点云）
- **点云查找表（LUT）**：像素到3D坐标的映射表

### 拍摄方法

与"标定板图像标定"类似，但需要额外准备：
1. 图像数据：标定板照片
2. 点云数据：`.pcd` 格式点云文件
3. LUT 数据：像素到3D的映射表 `.bin` 文件

### 拍摄注意事项

| ⚠️ 注意 | 说明 |
|---------|------|
| 需要手动配置 PCL 库支持 | 点云库支持需额外配置 |

| ✅ 推荐做法 | ❌ 避免做法 |
|-----------|-----------|
| 图像、点云、LUT 必须严格对应 | 数据不匹配 |
| LUT 制作需相机厂商提供工具 | 使用不兼容的 LUT |

### 数据提供方式（在 `calib.cpp` 中）

```cpp
// 图像文件列表
std::vector<std::string> img_files = {
    "../assert/1.png", "../assert/2.png",
    // ...
};

// 点云 LUT 文件列表
std::vector<std::string> cloud_luts_files = {
    "../assert/lut/1.bin", "../assert/lut/2.bin",
    // ...
};

// 点云 PCD 文件列表
std::vector<std::string> cloud_pcd_files = {
    "../assert/cloud/1.pcd", "../assert/cloud/2.pcd",
    // ...
```

## 常见问题与注意事项

### 常见问题

#### Q1：编译时找不到 Eigen3 或 OpenCV？

**A**：请确保已运行依赖安装脚本：

#### Q2：标定失败，RMS 误差过大？

**A**：可能原因：
- 标定板间距 `interval_mm` 设置不正确（请用卡尺实际测量）
- 图像数量不足或角度变化不够
- 标定板在图像中不完整
- 图像模糊或反光严重

**解决方案**：
1. 重新测量标定板间距并更新配置
2. 增加图像数量，确保足够的角度变化
3. 确保标定板完整出现在所有图像中
4. 改善拍摄环境，避免模糊和反光

#### Q3：3D 手眼标定旋转矩阵不准确？

**A**：**这是最常见的问题！**

**原因**：纯平移无法准确标定旋转矩阵

**解决方案**：
1. 确保每张图像都包含较大角度的旋转（±30°以上）
2. 避免规律性移动，增加随机性
3. 参考以下拍摄策略：

```
推荐拍摄序列：
1. 初始位置
2. 向前平移 + 绕 X 轴旋转 30°
3. 向后平移 + 绕 Y 轴旋转 -30°
4. 向左平移 + 绕 Z 轴旋转 45°
5. 向右平移 + 绕 X 轴旋转 -45°
6. 向上平移 + 绕 Y 轴旋转 30°
7. 向下平移 + 绕 Z 轴旋转 -30°
... 随机组合平移和旋转
```

#### Q4：RealSense 相机无法打开？

**A**：检查 USB 权限：

```bash
# 查看设备

# 重新加载权限规则

sudo udevadm trigger
```


#### Q6：相机内参标定和手眼标定哪个先做？

**A**：**内参标定必须先做！**

推荐操作顺序：
1. **先做内参标定** → 生成 `calib_intrinsic_1st.xml`
2. **再做 3D 手眼标定**（图像或球）→ 生成 `calib_extrinsic.xml`
3. **最后用 detect 验证** → 检查转换精度

#### Q7：如何选择标定方式？

| 场景 | 推荐标定方式 |
|------|------------|
| 2D 相机抓取定位 | 2D 4点标定或 2D 12点标定 |
| 3D 相机深度信息利用 | 3D 标定板图像标定 |
| 相机安装在机械臂末端（眼在手上） | 3D 标定板图像标定 |
| 相机固定在工作台（眼在手外） | 3D 标定板图像标定 |
| 需要极高精度 | 2D 12 点标定 |
| 快速标定 | 2D 4 点标定 |

---

### 重要注意事项总结

| 注意事项 | 说明 |
|---------|------|
| ⚠️ **先内参，后外参** | 内参标定必须在外参标定之前完成 |
| ⚠️ **标定板间距要准确** | 必须用卡尺实际测量，不要使用默认值 |
| ⚠️ **3D 标定必须包含旋转** | 纯平移无法准确标定旋转矩阵 |
| ⚠️ **图像要完整清晰** | 标定板必须完整出现在图像中，避免模糊 |
| ⚠️ **欧拉角约定要正确** | 根据机器人品牌设置正确的欧拉角顺序 |
| ⚠️ **验证标定结果** | 使用 detect 程序验证标定质量 |
