# ROS 2 桌面级 3D 目标检测与追踪系统技术报告

## 1. 项目目标与边界

本系统面向 Eye-in-Hand 机械臂场景，目标是在 ROS 2 Humble 下输出稳定的 3D 目标框与跟踪 ID，支持用户选择目标后进行机械臂跟随。

核心约束如下：

- 目标为可抓取物体，不强调细粒度分类。
- 输入为 RGB、Depth、CameraInfo 及 TF。
- 优先稳定性与抓取可用性，而非极限帧率。
- 支持手部/人体遮挡下的可用退化。


## 2. 系统实现架构

系统当前采用融合节点方案：检测与追踪在同一节点内执行，减少跨节点时延与状态丢失。

主要节点：

- `realsense2_camera_node`：提供 RGB/Depth/点云与相机 TF。
- `rgbd_detect_track_node`：融合感知主节点。
- `rviz_visualizer`：可视化转发与标注。
- `track_selector_node`：目标选择 UI。
- `following_node`：读取选中目标并驱动机械臂跟随。


## 3. 融合感知流水线

`rgbd_detect_track_node` 的处理流程可概括为：

1. RGB/Depth/CameraInfo 同步。
2. 基于图像时间戳查询 TF，失败时回退到最新 TF；若 TF 树不连通，可回退到源坐标系发布并标记 stale。
3. 图像分割输出目标掩码和人体掩码。
4. 执行 `object_mask - person_mask`，并做连通域主区域保留。
5. 深度投影为 3D 点，完成分位数裁剪、统计滤波、桌面剔除。
6. 计算 3D AABB，进行候选框帧内合并。
7. 3D 跟踪（含重复轨迹抑制、丢失轨迹漂移抑制）。
8. 发布兼容消息与扩展质量消息。


## 4. 分割后端可插拔设计

系统引入了 `segmentation` 抽象层，支持后端切换。

- 默认稳定方案：`yolov8_seg`
- 试验方案：`fastsam`

创建分割器示例（节选）：

```python
from robotic_follower.segmentation import create_segmenter_from_config

segmenter_cfg = {
    "type": "fastsam",
    "model_path": "FastSAM-s.pt",
    "person_model_path": "yolov8n-seg.pt",
}
segmenter = create_segmenter_from_config(segmenter_cfg, parent_node=self)
```

说明：

- FastSAM 为类无关分割，系统允许配置辅助 person 模型以增强“手/臂像素剔除”。
- 若 person 模型缺失，会退化使用形态学+最大连通域策略抑制粘连。


## 5. 关键工程改进

### 5.1 双框问题

根因拆分为两类：

- 可视化层叠加（原始检测框 + 追踪框同时显示）。
- 跟踪层重复建轨与重复输出。

改进措施：

- 默认关闭原始检测 Marker，仅显示追踪 Marker。
- 在跟踪器中加入近邻重复轨迹抑制。
- 在检测到追踪前新增候选框帧内合并（基于中心距离或 IoU3D）。


### 5.2 漂移问题

根因是丢失轨迹仍持续匀速外推。

改进措施：

- 对 `missing_count > 0` 的轨迹抑制位置外推，仅衰减速度。


### 5.3 TF 与日志噪声问题

改进措施：

- 支持 timestamp TF 查询失败时的最新 TF 回退。
- TF 树断开时支持回退到源坐标系发布，避免全链路停摆。
- 对高频告警做节流，提升调试可读性。


### 5.4 远距离目标过滤

新增过滤参数 `max_non_person_distance_m`（默认 1.2m）：

- 非人目标超出该距离直接剔除，减少远处误检干扰。


## 6. 消息接口

新增接口包 `robotic_follower_msgs`，核心消息：

```text
TrackedObject3D.msg
- tracking_id
- center
- size
- yaw
- score
- occlusion_ratio
- is_stale
- graspable
- label
```

同时保留兼容输出 `vision_msgs/Detection3DArray`，降低对下游模块改造成本。


## 7. 配置与运行

默认配置：

- `model/config/yolov8_seg_rgbd_track.yaml`

FastSAM 配置：

- `model/config/fastsam_rgbd_track.yaml`

启动示例：

```bash
ros2 launch robotic_follower perception_real.launch.py \
  fused_config_file:=model/config/fastsam_rgbd_track.yaml
```


## 8. 典型调试建议

- 先确认分割调试图是否正常刷新：`/perception/segmentation_debug_display`。
- 只保留追踪框可视化，避免“假双框”误判。
- 观察 `is_stale` 与 TF 告警，优先解决 TF 树连通与时间同步。
- 若仍有重复框，优先调 `detection_merge_dist_m` 与 `duplicate_track_dist_m`。


## 9. 结论

当前系统已实现“可运行、可调试、可扩展”的桌面级 3D 感知与跟随机制，具备以下工程特征：

- 架构上：融合感知，减少跨节点状态损失。
- 算法上：遮挡退化与去重策略完整。
- 工程上：可插拔分割后端与双消息输出兼容并存。
- 运维上：具备可视化诊断与告警节流能力。

可作为后续抓取闭环、任务编排与人机协作扩展的稳定基础版本。
